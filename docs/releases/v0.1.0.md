# CLAMP Core v0.1.0

First public release of the CLAMP runtime for Codex.

`CLAMP Core` packages the public, installable layer of the `Codex Local Augmented Memory Platform`: recall, theory-loop scaffolding, hot-context projection, task-end curation, and quiet task-start context assembly.

## Highlights

- One-command install for the public CLAMP runtime
- Pack-based architecture instead of ad hoc script copying
- Public-safe boundary: runtime only, no private memories or session history
- Five installable capability packs:
  - `ContextRecall`
  - `TheoryLoop`
  - `HotMemoryProjection`
  - `CurationLoop`
  - `QuietContext`
- Included public skills:
  - `clamp-core`
  - `clamp-maintenance`
- Isolated smoke-test CI so the repo can validate on a clean machine without private CLAMP data

## Install

```bash
git clone https://github.com/AaronShark/clamp-core.git
cd clamp-core
python3 install_clamp.py
```

Dry run:

```bash
python3 install_clamp.py --dry-run
```

## What This Release Gives You

- Local recall indexing over sessions, memories, and skills
- Explicit theory-loop notes for non-trivial engineering work
- Bounded hot-context generation
- Task-end curation and procedure staging
- Generated task briefs, focus briefs, and daily quiet-context artifacts

## Explicitly Not Included

This release does not ship:

- `~/.codex/memories/` contents
- `~/.codex/history.jsonl`
- `~/.codex/sessions/`
- `~/.codex/user-work-profile.md`
- personal captures
- private memory backup or full vault migration

`CLAMP Core` is the public runtime, not the private memory payload.

## Good Fit

Use this release if you want:

- a reusable Codex memory runtime
- a GitHub-installable CLAMP package for other Codex users
- a clean separation between public capability distribution and private memory sync

## Known Boundary

`v0.1.0` is intentionally scoped to the public runtime layer. Private cross-machine migration should remain a separate vault export/import workflow rather than being bundled into this repo.
